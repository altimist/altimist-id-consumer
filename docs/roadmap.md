# Roadmap

> **Purpose:** Map specs and epics to delivery windows (phases / sprints /
> releases). Orthogonal to epics (which group by domain) — this groups by
> when things ship.
> **Update when:** Phasing changes, new specs are added to a phase, or a
> phase ships.
> **Related:** [`specs/`](./specs/README.md), [`epics/`](./epics/README.md).

---

## Conventions

- **Phase** = a delivery window, typically multi-sprint. Each spec lists its
  phase in metadata.
- A phase usually spans multiple epics (cross-domain delivery).
- A phase ships when all its specs are merged and validated.
